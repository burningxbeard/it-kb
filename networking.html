Network Fundamentals 001

Describe each of layers of the OSI Model and how they fit into an Enterprise network?

Why is it important?

Include details how you would incorporate these layers into network troubleshooting

OSI Model

7. Application Layer

What it does: Provides the interface for network applications to interact with the network (e.g., web browsing, email, file transfers).

Key protocols:
  • HTTP/HTTPS (Hypertext Transfer Protocol / Secure): For web browsing.
  • SMTP (Simple Mail Transfer Protocol): For sending emails.
  • FTP (File Transfer Protocol): For transferring files.
  • DNS (Domain Name System): For resolving domain names to IP addresses.
6. Presentation Layer

What it does: Translates data between the application and the network, managing data encryption, compression, and formatting.

Key protocols:
  • SSL/TLS (Secure Sockets Layer / Transport Layer Security): For encrypting data in HTTPS.
  • JPEG, GIF: For image formats and data representation.
  • ASCII, Unicode: For text encoding.
5. Session Layer

What it does: Establishes, manages, and terminates sessions between applications on different devices.

Key protocols:
  • NetBIOS: Used for network sessions in local networks.
  • PPTP (Point-to-Point Tunneling Protocol): For VPNs and secure tunneling.
4. Transport Layer

What it does: Ensures reliable or fast delivery of data between devices, providing error detection and flow control.

Key protocols:
  • TCP (Transmission Control Protocol): Reliable, connection-based communication (used in web browsing, email).
  • UDP (User Datagram Protocol): Faster, but less reliable (used for video streaming, online gaming).

3. Network Layer

What it does: Routes data across different networks, using IP addresses for logical addressing and packet forwarding.

Key protocols:
  • IP (Internet Protocol): For routing and addressing data packets across networks.
  • ICMP (Internet Control Message Protocol): For network diagnostics (e.g., ping).
  • OSPF, BGP: For routing decisions.
2. Data Link Layer

What it does: Handles the transfer of data between adjacent network devices and error detection for frames.

Key protocols:
  • Ethernet: For wired LAN connections.
  • Wi-Fi (IEEE 802.11): For wireless LAN connections.
  • MAC (Media Access Control): For addressing devices on the network.
1. Physical Layer

What it does: Manages the physical connection between devices, including electrical signals, light pulses, and data transmission mediums.

Examples:
  • Ethernet cables, fiber optics, Wi-Fi signals.
  • Network Interface Cards (NICs).
OSI Model (Layers and Protocols
  • 7 - Application layer. Applications like FTP, web browsers, DNS, SNMP etc live here. Network services are requested here to be translated downstream over a network.
  • 6 - Presentation layer. Data representation and code formatting to ensure common format is used for sender and receiver to understand. Provides translation of data between the application and network layer. Can include protocols such as SSL/TLS for encryption.
  • 5 - Session layer. Establishes and manages the connections between networked computers/devices. For example, this layer is responsible for closing a session handled by TCP.
  • 4 - Transport layer. Ensures reliable transmission of data segments. Assembly/disassembly of data (segmentation/desegmentation), error control and flow control to control link reliability. TCP, UDP are protocols at this layer.
  • 3 - Networking. This is where routing takes place between different networks - each network has an address and this describes the process of routing packets from one node to another. Routing protocols, BGP, OSPF, RIP etc. Structure, logical addressing implemented here. NAT, HSRP, ICMP handled here.
  • 2 - Data Link layer. Node-to-node data transfer. MAC (Media Access Control) and LLC (Logical Link Control) for Ethernet, Wi-Fi, ZigBee. Also PPP (Point-to-point protocol). Flow control and data transmission reliability is defined here as well (CRC added to the frame). Switches work on this layer. ARP at this layer.
  • 1 - Physical layer. This is where the physical and electrical specifications are defined. Responsible for conversion of digital data into optical/electrical signals. Also include specification of pin layout, signal timing cable specs etc.

Network Fundamentals 002

Describe the TCP/IP Model and its layers?

How does it map to the OSI Model?
TCP/IP or Internet Model

It is a layered model that breaks down the network communication process into layers that carry out specific functions and was created/defined by DARPA (The Defense Advanced Research Projects Agency). It is also called the Internet or DoD Model.

Layers
  • 4 - Application layer. (process to process) Applications like FTP, web browsers, DNS etc live here. Provides applications to access services in the other layers.
  • 3 - Transport layer. (Host to host) This layer deals with connections between hosts. Protocols here are TCP and UDP for example.
  • 2 - Internet layer. (inter-networking) Responsible for host addressing, packaging and routing. Protocols such as ARP, IP, ICMP etc.
  • 1 - Network interface layer. LAN communication and physical/electrical specifications are defined here. Such as Ethernet, Token Ring etc.
Mapping (OSI⇒TCP/IP)
  • Application/Presentation/part of Session map to Application layer
  • Part of session/Transport map to Transport layer
  • Network maps to Internet layer
  • Data Link/Physical maps to Network Interface Layer
Network Fundamentals 003

Talk me through a network switch and its functionality.
Network Switch

Working at OSI Model layer 2 and sometimes layer 3 (When performing routing duties).

A device that filters and forwards frames between LAN segments. Separate collision domain for each port - i.e. each port can transfer data without interference/collisions. *Collision domain*

Multiple VLAN's can be configured on a switch - separate broadcast domains per VLAN. Without VLAN configured a single broadcast domain.

Switches can be interconnected via a crossover cable. Often switches have RJ45 ports but also SFP ports allowing different media to be used. Layer 3 specific features of a switch allow it to work like a router, Inter-VLAN routing and also routing protocols - depending on the switch or license.
Network Fundamentals 004

Talk me through a network router and its functionality.
Router

A device that forwards data packets between networks. A router is connected to at least two networks. Separate broadcast domains per interface as broadcast traffic is not forwarded.

Routing protocols and tables on OSI layer 3 to decide which connected network a packet should be sent. Router connects to ISP network allowing a route out but could be used internally also.

Firewall

A Network firewall is a Stateful firewall which monitors the state, protocol, port of the packet traversing the network from connection opening until it closes. Ensure only active connections are allowed to transit. Utilizes security rules such as ACL's to allow/deny traffic flows.

Network Fundamentals 008

Describe the term 'VLAN' and how this would be deployed in an enterprise network. Include overview of protocols and configuration.
VLAN on Enterprise Network

VLAN = Virtual Local Area Network

Understand VTP (VLAN Trunking Protocol), trunk links, routing between VLANs.

VLAN

It is any broadcast domain that is partitioned and isolated in a computer network at the data link layer (OSI layer 2). VLAN can be present over multiple switches (in different locations) in a network but still act like they are on the same LAN.

Trunk Links

IEEE 802.1Q or dot1q, networking standard defining VLAN tagging of ethernet frames. Used to allow multiple vlans to transit an interface to anther switch or router. Can prune vlans down particular trunks - switchport trunk allowed vlan {list}. Native VLAN is untagged, default vlan 1 but can be changed.

VTP - VLAN Trunking Protocol

VLAN Trunking Protocol. Cisco proprietary, Used to synchronize (add, delete, rename) VLANs across a layer 2 network. VTP Server where VLANs can be modified/created/deleted. VTP client synchronizes and forwards advertisements. A device in VTP transparent forwards advertisements only, VLANs are local. Each change increments revision number, need to make sure vlan database is clear when adding used switch to network to avoid higher revision number overwriting existing network with its stale configuration.

VLAN Routing

Options include multiple routers with vlans on different interfaces, router with sub interfaces, L3 switch.

  1. Configure VLAN on access interface to router. Define IP address on router for gateway.
  2. Have a single router and define a sub-interface for each VLAN. On the router Gi0/1.200 for VLAN 200 for example. Enable dot1q encapsulation for the right vlan number. Define IP address. On switch define trunk with dot1q encapsulation.
  3. L3 switch define a vlan interface on the switch with the IP address. Term for the interface is SVI = Switched Virtual Interface.

DHCP 001

Describe the DHCP process including its basic function and location in a network.

What other options does DHCP offer?
Dynamic Host Configuration Protocol

DHCP allows for the dynamic assignment of IP addresses so that devices can talk to other networks. Without DHCP server either static IP configuration is needed or self assigned APIPA (wont be able to talk outside of subnet).

DHCP can be configured on standalone servers (dhcpd, dnsmasq, dibbler, Windows Servers etc). Can also be configured on networking devices such as Routers.

UDP port number 67 is the destination port of a server, and UDP port number 68 is used by the client. DHCP Discover(Client) >> DHCP Offer(Server) >> DHCP REQUEST(Client) >> DHCP ACK(Server) >> probe for address (client) >> finalize lease(client).

DHCP offer includes subnet/gateway/dns server/dhcp server ip's/lease time etc. Client address probe ensures no duplicate IP's on subnet from multiple DHCP servers.

Options are additional parameters that a DHCP can send to the client. Including things such as NTP Server, hostname, domain name, tftp server, smtp server default ttl.

It helps in managing a large number of devices that require the use of IPs, very easily. IPAM (IP Address Management) is a tool that can help planning, tracking, and managing the IP Address spaces used in a network.
DHCP 002

Describe the different mechanisms for a client to keep the same IP address on a network.
DHCP Renew lease

DHCP address renewed by client sending DHCPREQUEST and server confirming with DHCPACK. DHCPACK including new lease information and updated parameters. If not renewed client needs to start lease generation process again.

DHCP Reservation

DHCP server reserves an IP address for a client based off the clients MAC address. This DHCP only uses the IP for this device.

Static IP address on a device

Statically define the IP information for the device. Minimum IP/valid netmask. Gateway needed for communication to external network.

DNS 005

What is Round Robin DNS and what benefits can is bring?
Round Robin DNS

Round Robin DNS is a method used for load distribution of DNS request against a domain name.

Round Robin DNS works by responding to DNS requests not only with a single potential IP address, but with a list of potential IP addresses corresponding to several servers that host identical services.

With each DNS response, the IP address sequence in the list is enumerated to the next in the list. The clients attempts a connection with the address returned from a DNS query, so on different connection request the clients would receive service from the next IP Address in the list.

Spanning Tree 002

What is Spanning Tree Protocol (STP) and how does this compare to Rapid Spanning Tree Protocol (RSTP)?
STP - Spanning Tree Protocol

STP is a layer 2 protocol that removes the possibilities of bridge loops within a LAN, regardless of the number of backup links within the bridged network. Exchanges BPDU (Bridge Protocol Data Units) messages with neighbor switches to detect and remove loops.

RSTP provides a faster "convergence" when topology changes occur. I.e. ports get to a forwarding state faster
Spanning Tree 003

Describe the different spanning states in STP and RSTP.
Spanning States
STP (802.1D) Port State Disabled A network administrator has manually disabled the switch port. Blocking The state of a port when not designated or root port. No frames forwarded. Default is 20secs when an interface needs to move away from blocking. Listening Only a root or designated port will move to the listening state. The non-designated port will stay in the blocking state. No data transmission occurs at this state for 15 seconds just to make sure the topology doesn’t change in the meantime. Learning At this moment the interface will process Ethernet frames by looking at the source MAC address to fill the mac-address-table. Ethernet frames however are not forwarded to the destination. It takes 15 seconds to move to the next state Forwarding The interface will forward Ethernet frames. RSTP (802.1w) Port State Discarding The state of a port when not designated or root port. Learning At this moment the interface will process Ethernet frames by looking at the source MAC address to fill the mac-address-table. Ethernet frames however are not forwarded to the destination. Forwarding The interface will forward Ethernet frames." "Spanning Tree 004 In Spanning Tree Protocol, what is meant by the following terms; 1) Edge port 2) Root port 3) BPDU Guard 4) Designated port 5) BPDU Filter" "Edge Port Port defined as the edge of the network. Used in RSTP to define an access port to speed to move from blocking to forwarding faster. Root Port The interface on a switch which connects either to the switch marked as the Root Bridge, or the next upstream switch towards the root bridge. BPDU Guard An STP setting which can set an interface to err-disabled when BPDU's are detected inbound on the switch. Often used on edge ports to protect against rogue switches. Designated port The interface on a switch which advertises the best path to the root bridge downstream. BPDU Filter An STP setting which will drop BPDU's so that they are not processed by the switch." "Spanning Tree 005 Within STP, describe the election of a root bridge and convergence up network changes?" "Root Bridge Has lowest Bridge ID, combination of priority and mac. Priority compared first, if tie lowest MAC wins. Election process involves initially sending out BPDU's indicating they are root, if a superior Bridge ID received from BPDU's switch readvertises better Bridge ID. Convergence Once Root Bridge found, switches decide best path to the root bridge based on the lowest cumulative path cost. Different link bandwidths have different path costs. Routing 001

What is HSRP (Hot Standby Routing Protocol)?
HSRP - Hot Standby Routing Protocol (Cisco proprietary protocol)

Is a first hop router redundancy protocol that allows routers to group together under a single default gateway. Should the active router fail a standby router would take the traffic.

A single virtual default gateway is assigned on all routers. Router with highest priority in a standby group will be the active router. A single interface MAC is used with HSRP by default 0000.0c07.ac** (where ** is the HSRP group number in hex).

Only active router is sending and receiving traffic. Uses multicast UDP 1985 for IPV4, 2029 for IPV6 for HSRP status traffic for the interface. When active is detected down, next highest priority router takes the traffic and becomes new active. HSRP reduces the priority upon link down detections.

Routing 003

How does distance vector work and which protocols use it?
Distance Vector Routing

Distance-vector routing protocols measure the distance by the number of routers a packet has to pass, one router counts as one hop. Routes shared regularly between routers via broadcasts.

.This is used by RIP, IGRP and EIGRP
Routing 004

How does Link state work and which protocols use it?
Link State & Protocols

Link state knows the same cost as Distance Vector (number of routers a packet has to pass) as well as the speed of the link and whether that link is up/down.

Each router calculates shortest path to each other router using that cost. Notable difference from distance vector routing is that routes on changes.

This is used by OSPF (Open Shortest Path First) and IS-IS (Intermediate System to Intermediate System).

Network General 005

What is VPN?
VPN - Virtual Private Network

Allows a secure tunnel between devices which encapsulates and encrypts traffic to secure connectivity between networks.

Remote access VPN to securely connect device (like laptop) to corporate VPN. Site-to-site VPN connecting two corporate offices, or FC to DC etc.

Common protocols: IPSec, OpenVPN, GRE, IP in IP, SSTP etc.
Network General 006

What is VRF?
Virtual routing and forwarding

VRF creates a separate routing table for each VRF to allow network segmentation without additional hardware. After VRF created, assign the VRF to an interface (vlan etc), then the routes are held in that VRF.
Network General 007

What is an ACL?
ACL - Access Control List

A set of rules which defines the traffic flow through an interface. Basic ACL's utilise source/destination IP's whereas Extended ACL's can include information on the protocol/port number, connection state etc as well as source/destination addresses. Can be applied into and out of an interface including SVI's.
Network General 008

What is SNMP?
SNMP - Simple Network Management Protocol

It is a network protocol used for collecting organizing and exchanging information between network devices. SNMP is widely used in network management for configuring network devices like switches, hubs, routers, printers, servers. Object ID's (OID's) are defined for specific devices for particular outputs/inputs - Management Information Base (MIB) files holds database of these. Community strings are used to authenticate the request.
