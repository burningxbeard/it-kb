Active Directory 001

What is Active Directory?
Microsoft - Active Directory

An Active Directory domain is a collection of objects within a Microsoft Active Directory network.

An object can be a single user, a group or it can be a hardware component, such as a computer or printer.

Each domain holds a database containing object identity information.

The main service in AD is Domain Services (AD DS), which stores directory information and handles the interaction of the user with the domain.

AD DS verifies access when a user signs into a device or attempts to connect to a server over a network.

Domain Services controls which users have access to each resource.

Active Directory 004

What are the Active Directory Services and describe their function?
Microsoft - Active Directory - Services

Active Directory Certificate Services (AD CS)

* Provides customizable services for issuing and managing digital certificates used that employ public key technologies.

* These can be used to encrypt and digitally sign electronic documents and messages.

* These digital certificates can be used for authentication of computer, user, or device accounts on a network.

* Digital certificates are used to provide:

* Confidentiality through encryption

* Integrity through digital signatures

* Authentication with computers, users, or device accounts within a domain

Active Directory Domain Services (AD DS)

* AD DS stores information about user accounts, such as names, passwords, phone numbers, etc...

* It enables authorized users on the same network to access this information.

* It stores information about objects on the network

* Makes this information easy for administrators and users to find and use.

* These objects typically include shared resources such as servers, network shares, printers.

Active Directory Federation Services (AD FS)

* Enables Federated Identity and Access Management by securely sharing digital identities and entitlements rights.

* Extends the ability to use single sign-on functionality within the network to external facing applications.

Active Directory Lightweight Directory Services (AD LDS)

* Is a Lightweight Directory Access Protocol (LDAP) directory service that provides support for directory enabled applications

* Does not need the dependencies and domain restrictions of Active Directory Domain Services (AD DS) Server.

Active Directory Rights Management Services (AD RMS)

* AD RMS allows users and admins via Information Rights Management (IRM) policies to specify access permissions to documents.

* This prevents sensitive information from being accessed, printed, forwarded or copied by unauthorized users.

* Permission for a file are enforced no matter where the information is held, as the permission for the file is within the document.

Active Directory 012

What is DFS?
Microsoft - Active Directory - Distributed File System

Client →Server services

Distributed File System (DFS) is used to organize many file shares into a distributed file system.

DFS has two components to its service: Location transparency (via the namespace component) and Redundancy (via the file replication component).

Together, these components improve data availability in the case of failure or heavy load by allowing shares in the same or multiple different locations to be logically grouped under one folder, the "DFS root". This makes it possible to give users a virtual view of shared folders, where a single path leads to files located on multiple servers.

File System 001

What is the advantage of NTFS over FAT32?
Microsoft - File System - NTFS vs. FAT32

NTFS was build for better performance, especially when it comes to managing the way files are stored and retrieved.
* NTFS supports encryption
* NTFS supports compression
* NTFS supports improved metadata support.
* NTFS is more robust, secure and has better utilization of hard drive space.
* NTFS is Fault Tolerance an can automatically repair files/folders in the case of power failures or errors. FAT32 maintains two different copies of the FAT in the case of damage.
* NTFS Supports long file name of up to 255 characters
* NTFS support file and folder permissions, while FAT32 only supports folder permissions, does not support file permissions
* NTFS allows Partitions up to 256TB, and single files of 256TB
* FAT32 allows partitions max of 32GB, single files up to 4GB

Group Policy 001

What is a Group Policy?
Microsoft - Group Policy

Group Policy is a feature of Microsoft Windows Active Directory that adds additional controls to user and computer accounts.

Group Policies provide centralized management and configuration of computer and user accounts within an Active Directory Domain environment.

Group Policy allows the application of policies to be deployed domain-wide or to an individual set of user and computer accounts.

Organizational Units and Security Groups can all be used to allow policies to only be deployed to the correct set of user and computer accounts.

Windows Server

What is Shadow Copy?
Microsoft - Shadow Copy

AKA: Volume Snapshot Service, Volume Shadow Copy Service or VSS

Shadow Copy is a technology included in Microsoft Windows that allows taking manual and automatic backup copies or snapshots of files and volumes.

Shadow Copy is also capable of creating backups or snapshots of computer files or volumes, even when they are in use.

Shadow Copy is able to support a primitive method of file versioning history, by allowing any file to be retrieved as it existed at the time any of the snapshots was made.

Shadow Copy requires either NTFS or the ReFS file system to create and store shadow copies.

Shadow Copy can create copies can be created on local and external (removable or network) drives/volumes for backup and recovery operations.
Windows Server

How does Shadow Copy operate?
Microsoft - Shadow Copy / Volume Snapshot Service

AKA: Volume Snapshot Service, Volume Shadow Copy Service or VSS Shadow Copy operates at the block level for volumes. This snapshot is a read-only point-in-time copy of the volume. Snapshots allow the creation of consistent backups of a files/volumes, ensuring that the contents do not change and are not locked while the backup is being made. This allows the system to maintain operational during the process.
